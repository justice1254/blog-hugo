정적 웹사이트의 모든 것: Hugo와 Azure SWA 심층 가이드

정적 웹은 단순히 옛날 기술이 아닙니다. CDN·서버리스·CI/CD 환경과 결합하면서 가장 강력하고 효율적인 배포 방식 중 하나로 부활했습니다. 이 글에서는 Hugo 기반의 사이트를 Azure Static Web Apps에 배포하는 전 과정을 세밀하게 다룹니다.

1. 정적 웹의 장점

  • 속도: 서버 렌더링 없이 CDN에서 바로 서빙 → TTFB 최소화.
  • 보안: DB나 동적 코드 없음 → 공격 표면 대폭 감소.
  • 비용: 서버 유지비 제로, 무료 티어로도 충분히 운영 가능.

2. Hugo의 구조

/
├─ content/     # 글/페이지
├─ layouts/     # 템플릿
├─ static/      # 이미지/JS/CSS
├─ themes/      # 테마
└─ config.toml  # 사이트 설정

3. Azure SWA 배포 전략

  1. GitHub 레포 연결 → Actions 자동 생성.
  2. 빌드: hugo --gc --minify, 출력은 public.
  3. SWA에서 app_location을 public으로 지정.

4. 성능 최적화

항목권장효과
이미지WebP/AVIF 변환용량 50% 이상 절감
폰트preload + font-display:swapCLS 방지
JS필요 최소화, defer렌더링 차단 요소 감소

5. SEO & 접근성

  • title/description 메타 충실히 작성.
  • Open Graph/Twitter 카드 이미지 제공.
  • 색상 대비와 heading 계층 구조 준수.

6. 자주 발생하는 문제와 해결

❌ 글이 목록에 안 보임

대부분 draft=true 또는 미래 시각 설정 때문. 과거 시각 + draft=false로 저장하세요.

❌ 빌드 성공했는데 화면이 비어있음

public/이 비어 있거나 테마 목록 템플릿이 글을 숨김. 로컬 빌드로 먼저 확인.
